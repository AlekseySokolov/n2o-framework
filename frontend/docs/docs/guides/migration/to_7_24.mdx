## Изменения в API автотестов {#changes-autotest-api}

- Перешли на Selenide версии 6.6.0
- Все названия методов проверок состояния элементов приведены к общему виду `shouldHave*`, `shouldBe*`
- Логика введение значений в поле ввода была изменена. Теперь чтобы ввести значение в поле нужно вызвать отдельно методы `click`, `clear` и `setValue`.
Метод `val` был переименован на `setValue`.
- Теперь интерфейс `ButtonField` наследуется от `Button`
- Метод содержащие атрибут `String regex` были приведены к виду `*Matches(String regex)`.
